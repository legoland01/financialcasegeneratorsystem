# 提示词0.5：证据归属规划表生成

## 版本信息
版本: v1.0.0-alpha
最后更新: 2026-01-19

## 任务背景
本任务将判决书提及的所有证据进行归属规划,明确每个证据应该由哪方提交,为后续生成原告和被告证据包提供依据。

## 输入参数

### 必填参数
- `0.1_结构化提取结果`: JSON对象,来自提示词0.1的输出
- `0.3_交易时间线`: JSON数组,来自提示词0.3的输出
- `0.2_Profile库`: JSON对象,来自提示词0.2的输出

## 输出要求
输出必须为标准JSON格式,包含以下字段:

```json
{
  "证据归属规划表": [
    {
      "证据序号": "integer",
      "证据名称": "string",
      "应归属方": "string (原告/被告/双方/法院调取)",
      "文件类型": "string (合同/凭证/登记/文书/表格/其他)",
      "是否需要生成": "boolean",
      "关键数据提示": {
        "涉及金额": {
          "数值": "number",
          "单位": "string"
        },
        "涉及日期": "string (YYYY-MM-DD)",
        "涉及方": ["string"],
        "其他": "string"
      },
      "关联交易节点": "integer (关联交易时间线的序号)",
      "证据组": "integer (同一组证据的编号)",
      "证明目的": "string"
    }
  ],
  "证据分组": {
    "证据组_1": {
      "组名称": "string",
      "归属方": "string",
      "证据数量": "integer",
      "证明目的": "string"
    }
  }
}
```

## 执行步骤

### 步骤1: 扫描判决书识别证据
- 扫描判决书,识别所有提及的证据
- 提取证据名称

### 步骤2: 确定证据归属
- 根据证据内容和性质,判断应该由哪方提交
- 原告提交的证据:支持原告诉讼请求的证据
- 被告提交的证据:支持被告抗辩意见的证据
- 双方提交的证据:双方都可能提交的证据
- 法院调取的证据:法院依职权调取的证据

### 步骤3: 分类证据文件类型
- 合同类:融资租赁合同、担保合同等
- 凭证类:银行回单、发票等
- 登记类:抵押登记、质押登记等
- 文书类:催款函、律师函等
- 表格类:设备清单、租金表等
- 其他:不属于上述类型的证据

### 步骤4: 判断是否需要生成
- 法院采信的证据:需要生成
- 法院不予采信的证据:可能需要生成(如被告提交但未采信的证据)
- 仅提及未展开的证据:根据判断决定是否生成

### 步骤5: 提取关键数据提示
- 提取证据涉及的金额
- 提取证据涉及的日期
- 提取证据涉及的各方
- 提取其他关键信息

### 步骤6: 关联交易节点
- 根据证据内容,关联到交易时间线中的相应事件
- 确保证据与交易事实对应

### 步骤7: 进行证据分组
- 将相关证据归入同一证据组
- 为每个证据组定义名称和证明目的

### 步骤8: 验证完整性
- 检查判决书所有提及的证据是否都在表中
- 检查证据归属是否合理
- 检查证据分组是否合理

## 质量控制检查点

### 即时检查(嵌入生成过程)
- ✅ 判决书所有提及的证据都在表中
- ✅ 证据归属明确
- ✅ 文件类型分类正确
- ✅ 是否需要生成判断合理

### 数据一致性检查
- ✅ 证据名称与判决书一致
- ✅ 涉及方与Profile库一致
- ✅ 关联交易节点存在

### 逻辑合理性检查
- ✅ 原告证据支持诉讼请求
- ✅ 被告证据支持抗辩意见
- ✅ 证据分组逻辑合理

### 异常处理
- ⚠️ 如果证据归属不明确,根据证据内容推断并标注
- ⚠️ 如果证据类型不明确,归入"其他"类型

## 执行指令
请仔细扫描判决书,识别所有提及的证据,明确每个证据的归属、类型、是否需要生成、关键数据和关联交易节点。确保证据归属规划完整、合理、逻辑清晰。

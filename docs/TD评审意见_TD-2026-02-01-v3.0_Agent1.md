### 2.3 具体修改建议

**修改位置**: TD第3.1节"整体架构图"的main.py说明

**当前TD内容**（第61-65行）：
```markdown
│  main.py                                    # 统一入口                        │
│  ├── --input PATH                          # 输入判决书路径                   │
│  ├── --output PATH                         # 输出PDF路径                     │
│  ├── --verify                              # 仅验证模式                      │
│  └── --config PATH                         # 配置文件路径                     │
```

**建议修改为**：
```markdown
│  main.py / FinancialCaseGenerator    # v3.0统一入口（废弃v2.x的run_complete.py）
│  ├── --input PATH                    # 输入判决书路径
│  ├── --output PATH                   # 输出证据集PDF路径
│  ├── --case-type TYPE                # 案件类型（可选）
│  └── --verify                        # 仅验证模式
│
│  v3.0数据流：
│  判决书 → F2.1案情分析 → F2.2诉求提取 → F2.3证据规划 → F2.4证据收集 → F2.5证据列表 → LLM生成 → PDF输出
```

**建议添加位置**: 第3.1节末尾或作为新节（如3.1.1）

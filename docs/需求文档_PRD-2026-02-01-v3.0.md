# 金融案件证据集生成系统 v3.0 - 产品需求文档

**文档编号**: PRD-2026-02-01-v3.0
**版本**: v3.0.0
**日期**: 2026-02-01

---

## 一、产品概述

### 1.1 产品目标

输入一份判决书PDF，输出一份符合法院标准的证据集PDF。

### 1.2 用户故事

| 角色 | 场景 | 需求 |
|------|------|------|
| 法官 | 收到融资租赁案件判决书 | 快速生成完整的证据提交材料 |
| 律师 | 代理融资租赁案件 | 整理证据材料，生成标准化的证据集 |
| 当事人 | 需要提交证据材料 | 一键生成符合法院要求的证据文档 |

### 1.3 产品愿景

**告诉系统要什么，系统就生成什么。**

---

## 二、系统边界

### 2.1 输入输出

```
输入: 判决书 PDF
输出: 证据集 PDF（包含起诉状 + 证据材料）
```

### 2.2 功能范围

| 功能 | 说明 |
|------|------|
| 判决书解析 | 理解判决书内容，提取关键信息 |
| 证据生成 | 根据判决书内容生成证据材料 |
| PDF输出 | 生成符合法院标准的证据集 |

### 2.3 非功能范围（不包含）

- 判决书内容生成
- 法院系统对接
- 电子送达

---

## 三、功能清单

### 3.1 判决书解析功能

| 功能 | 说明 |
|------|------|
| F1.1 文本提取 | 从PDF判决书中提取文本内容 |
| F1.2 当事人识别 | 识别并提取案件当事人信息 |
| F1.3 事实提取 | 从判决书中提取案件事实 |
| F1.4 诉讼请求提取 | 从判决书中提取诉讼请求 |
| F1.5 金额提取 | 从判决书中提取金额、日期等关键数字 |

### 3.2 证据生成功能

| 功能 | 说明 |
|------|------|
| F2.1 起诉状生成 | 根据诉讼请求生成民事起诉状 |
| F2.2 合同生成 | 根据判决书涉及的合同生成合同文本 |
| F2.3 凭证生成 | 根据判决书涉及的支付生成凭证 |
| F2.4 文书生成 | 根据判决书涉及的文书生成相应文本 |
| F2.5 附件生成 | 生成合同相关的租赁物清单、租金计划等附件 |
| F2.6 证据索引生成 | 生成证据材料清单和索引 |

### 3.3 PDF输出功能

| 功能 | 说明 |
|------|------|
| F3.1 文本渲染 | 将证据文本渲染为PDF格式 |
| F3.2 表格渲染 | 将附件表格渲染为PDF表格 |
| F3.3 分页处理 | 正确分页，每份证据独立成页 |
| F3.4 证据集打包 | 将所有证据打包为单一PDF文件 |

## 四、质量标准

### 4.1 内容质量

| 标准 | 要求 |
|------|------|
| 真实性 | 证据内容必须与判决书一致 |
| 完整性 | 判决书中涉及的证据都必须生成 |
| 准确性 | 当事人信息、金额、日期必须准确 |

### 4.2 格式质量

| 标准 | 要求 |
|------|------|
| 脱敏标记 | 证据中不允许出现脱敏标记（如"某某公司"） |
| 格式规范 | 符合法院对证据材料的标准格式要求 |
| 证据编号 | 证据编号唯一，顺序合理 |

### 4.3 数据一致性

| 检查项 | 要求 |
|--------|------|
| 金额一致 | 合同金额与附件金额一致 |
| 当事人一致 | 同一当事人的名称在所有证据中一致 |
| 日期一致 | 同一事件的日期在所有证据中一致 |

---

## 五、验收标准

### 5.1 功能验收

| 验收项 | 标准 |
|--------|------|
| 输入支持 | 支持PDF格式的判决书输入 |
| 输出完整 | 包含起诉状和所有证据材料 |
| 格式正确 | 输出为符合法院标准的PDF |

### 5.2 质量验收

| 验收项 | 标准 |
|--------|------|
| 无脱敏标记 | 证据材料中无"某某公司"等脱敏标记 |
| 内容准确 | 证据内容与判决书一致 |
| 格式规范 | 符合法院标准格式 |

---

## 六、约束条件

### 6.1 设计约束

| 约束 | 说明 |
|------|------|
| 脱敏标记隔离 | Prompt中不出现脱敏标记，只使用真实名称 |
| 要素驱动 | 系统提取判决书要素，由LLM生成内容 |
| 简洁指令 | 告诉LLM要生成什么，不告诉它怎么做 |

### 6.2 性能约束

| 约束 | 说明 |
|------|------|
| 响应时间 | 单次生成请求在合理时间内返回 |
| 资源使用 | 系统资源消耗在合理范围内 |

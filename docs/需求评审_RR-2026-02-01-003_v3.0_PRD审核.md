## 二、问题与建议

### 问题1：附件规划缺失 ⚠️ P0

**位置**: 3.1节流程图、4.2节F2.10

**问题描述**：
- 流程图中没有明确"附件是独立文件还是正文包含"
- 合同生成后，租赁物清单作为附件没有规划
- 示例Prompt说"租赁物：详见附件1租赁物清单"，但没有说明附件如何生成

**建议**：

在F2.3证据规划中增加附件规划逻辑：
```
F2.3 证据规划
├─ 根据诉求确定需要证明的事实
├─ 根据案件性质（合同类型）确定证据类型
├─ 规划每项诉求需要哪些证据支撑
└─ 规划每份证据的附件形式（独立文件/正文包含/不需附件）
```

### 问题2：证据列表结构不完整 ⚠️ P1

**位置**: 第355-360行

**问题描述**：
证据列表中"附件类"的描述不够明确：
```
| 附件类 | 序号、名称、数量/金额（表格形式） | 标的明细 |
```

**建议**：

完善证据列表结构：
```markdown
| 附件类 | 附件类型（租赁物清单/还款计划）、数据来源、生成方式（独立文件/正文包含） | 标的明细 |
```

### 问题3：LLM示例Prompt有脱敏标记 ⚠️ P1

**位置**: 第232-268行

**问题描述**：
示例中还有"XXX"和"X月X日"等占位符

**建议**：
更新示例，使用真实数据或明确标注"此处应填充从案情基本数据集提取的值"

### 问题4："自行编造"规则不清晰 ⚠️ P2

**位置**: 3.2.4节、第208行

**问题描述**：
"自行编造缺失但支撑诉求必须的证据"没有说明编造范围

**建议**：
在F2.4中增加"自行编造规则"：
```
4. 自行编造缺失但支撑诉求必须的证据
   - 可编造：设备名称、型号、数量（根据案件类型配置）
   - 不可编造：当事人信息、金额、日期（必须来自案情基本数据集）
   - 需验证：编造的数据需符合案件逻辑
```
